2. 바탕화면에 새폴더 만들어서 폴더 안에서 'git bash here' 누르기
3. 'git clone'+깃허브 링크 작성 : 깃허브에 저장된 프로젝트를 다운로드 함


**PUSH
깃허브에 소스를 업로드 하는 과정
1. add (플러스 버튼) : 추가
2. comment 작성
3. commit : 저장
4. push : 깃허브 저장소에 코드 업로드 

**PULL
깃허브에서 소스 작성후 
vs에서 pull-> pull from -> origin main

퇴근 직전에 push (충돌할 수 있기 때문)
출근 하자마자 pull

GIT => public 색깔 강함
SVN => private 색깔 강함 

**팀프로젝트 하기 
깃허브=> 설정 => add people => 받는 사람 이메일 

**pull을 먼저 하지 않고 작업을 시작하면 
팀원 소스와 내 소스가 섞인다. 
소스가 길어질수록 심하게 섞인다.
merge 작업을 통해 해결해야함

소스가 없어지거나 너무 많이 섞여서 해결하기 힘들 때 
깃허브 히스토리를 확인하자!
히스토리를 확인하기 위해서는 (코멘트)를 명확하게 작성하자
